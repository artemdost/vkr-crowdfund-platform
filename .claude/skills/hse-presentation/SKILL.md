# HSE Presentation Skill — строгое следование шаблону НИУ ВШЭ

## Когда применять
Любой запрос на создание, изменение, пересборку или добавление слайдов к презентации ВКР: «сделай презентацию», «добавь слайд», «поменяй формулировку на слайде», «собери презентацию заново», «сгенерируй pptx».

## ЖЁСТКИЕ ПРАВИЛА (нарушение = ошибка)

### ❌ Что ЗАПРЕЩЕНО
1. **Добавлять любые элементы, которых нет в шаблоне**: декоративные линии, иконки, градиенты, фоновые картинки, ленточки, дополнительные логотипы.
2. **Менять цвета шаблона**. Палитра жёстко фиксирована (см. ниже).
3. **Менять шрифты**. Только HSE Sans (fallback Calibri).
4. **Менять позиции и размеры shape'ов header'а**. Координаты всех элементов жёстко зафиксированы в шаблоне.
5. **Использовать собственные дизайн-решения**, если они не воспроизводят один из 12 layout'ов.
6. **Добавлять footer от себя** (номер слайда, подпись) — всё это уже есть в master slide.
7. **Создавать pptx с нуля**. Только через копирование layout'ов из `template.pptx`.

### ✅ Что РАЗРЕШЕНО
1. Заполнять placeholder'ы текстом (с заменой plaintext-подсказок типа «Название презентации»).
2. Вставлять изображения в соответствующий image-placeholder layout'а.
3. Дублировать слайды, использующие один и тот же layout.
4. Использовать цвета из палитры (см. ниже) для акцентов внутри текста.

## Шрифты
- **Основной:** HSE Sans
- **Fallback:** Calibri Light (заголовки), Calibri (текст)
- **Размеры (из шаблона):**
  - Заголовок презентации (обложка): 43 pt
  - Заголовок слайда: 28-32 pt
  - Подзаголовок: 18 pt
  - Основной текст: 13 pt
  - Подписи в header'е: 10 pt
  - Цифры на layout «Цифры»: 70+ pt

## Геометрия
Формат: 13.33" × 7.50" (16:9), 33.87 × 19.05 cm.

## Технический процесс генерации

Рекомендованный инструмент: **`python-pptx`** с загрузкой `template.pptx` и использованием существующих layout'ов через `slide_layouts`.

**НЕ НАДО:**
- Создавать `Presentation()` без template
- Использовать онлайн-редакторы, у которых нет возможности загрузить шаблон (Claude Design может потерять структуру)
- Конвертировать из markdown напрямую (теряется шаблон)

**Схема работы:**

## Проверка результата (QA)

После генерации `presentation.pptx` — выполнить проверку соответствия шаблону:
1. Открыть получившийся файл в PowerPoint или Keynote.
2. Убедиться, что количество slide master'ов не увеличилось (должен быть 1).
3. Убедиться, что количество layout'ов не увеличилось (должно остаться 12).
4. Убедиться, что header каждого content-слайда идентичен шаблону по структуре.
5. Убедиться, что добавленных shape'ов нет (только placeholder'ы).
6. Убедиться, что шрифт и цвета соответствуют палитре.

Если любое из условий нарушено — **не выдавать результат пользователю, а пересобрать** через builder.

## Важные принципы

1. **Template-first.** Сначала открыть шаблон, потом наполнить. Никогда не наоборот.
2. **No creativity with design.** Дизайн уже сделан университетом. Задача — заполнить placeholder'ы.
3. **Идемпотентность.** Пересборка с тем же `content.yaml` должна давать побайтово идентичный (или почти) результат.
4. **Валидировать до выдачи.** Если pptx сгенерирован с нарушениями — починить, не просто предупредить.

# Skill: hse-presentation

Локальный скилл для генерации презентации ВКР строго в шаблоне НИУ ВШЭ.

## Файлы

- `SKILL.md` — инструкция для Claude (правила и ограничения)
- `template.pptx` — официальный шаблон НИУ ВШЭ (копия из Референсов)
- `content.yaml` — контент 7 слайдов презентации ВКР
- `build_presentation.py` — Python-скрипт, собирает pptx из шаблона + контента
- `README.md` — этот файл

## Как собрать презентацию

1. При необходимости отредактируйте `content.yaml` — только значения полей, не структуру.
2. Запустите из корня репозитория:
   ```bash
   python3 .claude/skills/hse-presentation/build_presentation.py
   ```
3. Результат: `presentation/ВКР_презентация.pptx`

## Что делает скрипт

1. Открывает `template.pptx` (официальный шаблон ВШЭ).
2. Удаляет 16 демо-слайдов, оставляя мастер и 12 layout'ов.
3. Создаёт новые слайды **только через** `slide_layouts[name]` — структура шаблона не ломается.
4. Заполняет placeholder'ы контентом из `content.yaml`.
5. Валидирует: 1 мастер, 12 layout'ов, имена layout'ов совпадают.
6. Сохраняет результат.

## Что нельзя трогать

- `template.pptx` — никогда не модифицируется, только читается.
- Имена layout'ов в `content.yaml` — только из списка: `Обложка`, `Текст_1`, `Текст_2`, `Текст_3`, `График_1`, `График_2`, `Цифры`, `Таблица_1`, `Таблица_2`, `чистый_2`, `чистый` (и `цвет` как служебный).
- `build_presentation.py` — менять можно только механизмы заполнения, не архитектуру «загрузи template → добавь слайд через layout».

## Ограничения текущей версии

- Вставка внешних изображений (через `image_url` в content.yaml) пока не реализована. После сборки открыть pptx в PowerPoint/Keynote и подставить картинку вручную, или дополнительная логика через Claude Design.
- Таблица на слайде 6 (Таблица_2) может потребовать ручной корректировки количества строк/колонок в pptx.

## При изменении контента

1. Правьте `content.yaml` (только значения).
2. Перезапустите `build_presentation.py`.
3. Откройте `presentation/ВКР_презентация.pptx`, проверьте визуально.
4. Закоммитьте изменения yaml + pptx.

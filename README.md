## Прототип

Рабочий прототип развёртывается локально без Docker.

### Что реализовано

- 3 смарт-контракта: **InvestToken** (ERC-1155), **FundFactory**, **CrowdFund**
- Конечный автомат кампании: Funding → Active → Completed/Failed
- Milestone-эскроу с голосованием ERC-1155 токенами
- Автоматическое распределение бюджета при одобрении этапа (в той же транзакции)
- Пропорциональный возврат средств через pull payments при провале
- React-фронтенд с подключением MetaMask через RainbowKit
- Express API + SQLite для off-chain метаданных
- Event Listener синхронизации с блокчейном

### Быстрый запуск

Требования: Node.js 20 (Node 22+ не поддерживается Hardhat 2).

```bash
cd prototype

# 1. Установка
npm install
cd backend && npm install && cd ..
cd frontend && npm install && cd ..

# 2. Запуск (в 3 терминалах)
# Терминал 1: локальный блокчейн
npx hardhat node

# Терминал 2: развёртывание + backend
npx hardhat run scripts/deploy.js --network localhost
cd backend && node server.js

# Терминал 3: frontend
cd frontend && npm run dev
# → http://localhost:5173
```

### Демо-наполнение

Скрипт `scripts/seed.js` создаёт три тестовых кампании в разных состояниях:
- Funding (сбор средств, 35% собрано)
- Active (цель достигнута, голосование по этапу 1)
- Completed (все этапы одобрены, средства выплачены)

```bash
npx hardhat run scripts/seed.js --network localhost
node scripts/seed_backend.js   # связывает DB-записи с on-chain контрактами
```

### Покрытие тестами

- **94%** по строкам
- **88%** по ветвям
- Сквозной сценарий: 3 инвестора × 3 этапа с одним отклонением и повторной подачей

## Текст ВКР

### Сборка docx из markdown

```bash
cd thesis
python3 build.py    # создаст ВКР_полный.docx
```

Требования: `python-docx`, `lxml`.

### Оформление

Текст оформлен по ГОСТ 7.32-2017 и Правилам ВКР ПИ (11.2020):
- Times New Roman, 14 pt, межстрочный 1,5
- Поля: левое 3 см, правое 1,5 см, верхнее/нижнее 2 см
- Заголовки глав: 16 pt, жирный, с новой страницы
- Приложение: «ПРИЛОЖЕНИЕ А» по центру + тематический заголовок

### Структура

- Введение (~5 стр)
- Глава 1. Анализ предметной области и выявление проблем
- Глава 2. Проектирование платформы токенизированного инвестирования
- Глава 3. Архитектура и реализация прототипа платформы
- Заключение
- Библиографический список
- Приложение А — Экранные формы прототипа
